### 활동 2.1: t=3, n=100일 때 직접 트리 그려보기

**B-Tree 속성 복습 (t=3)**:
- 모든 노드는 최소 `t-1 = 2`개 키
- 모든 노드는 최대 `2t-1 = 5`개 키
- 루트는 예외적으로 최소 1개 키 가능

**최소 키 수로 트리 구성 (최대 높이)**:
```
높이 0 (루트): 1개 키
높이 1: 2개 노드 × 2개 키 = 4개 키
높이 2: 4개 노드 × 2개 키 = 8개 키
높이 3: 8개 노드 × 2개 키 = 16개 키

총 키 수: 1 + 4 + 8 + 16 = 29개 (높이 3)
```

**최대 키 수로 트리 구성 (최소 높이)**:
```
높이 0 (루트): 5개 키
높이 1: 6개 노드 × 5개 키 = 30개 키
높이 2: 36개 노드 × 5개 키 = 180개 키

총 키 수: 5 + 30 + 180 = 215개 (높이 2)
```

**결론**: n=100일 때, 높이는 2 또는 3

### 활동 2.2: 디스크 접근 횟수 계산

**시나리오 설정**:

- 각 노드 = 1개 디스크 블록

```

```
[데이터 정리]

```

## 학습 활동

### 활동 3.1: "n개의 키를 가진 B-Tree의 최대 높이는?" 질문에 답하기

**문제 정의**:
```
주어진 조건:
- B-Tree의 최소 차수 = t
- 총 키 수 = n

질문: 최대 높이 h는?
```

**사고 과정**:

1. **최소 키 수 트리 구성** (최대 높이를 위한 조건)
   ```
   높이 0 (루트): 최소 1개 키
   높이 1: 최소 2(t-1)개 키
   높이 2: 최소 2t(t-1)개 키
   높이 k: 최소 2t^(k-1)(t-1)개 키
   ```

2. **총 키 수 합산**:
   ```
   n ≥ 1 + 2(t-1) + 2t(t-1) + ... + 2t^(h-1)(t-1)
   
   = 1 + 2(t-1)[1 + t + t² + ... + t^(h-1)]
   
   = 1 + 2(t-1) × (t^h - 1)/(t - 1)
   
   = 1 + 2(t^h - 1)
   
   = 2t^h - 1
   ```

3. **높이 h 도출**:
   ```
   n ≥ 2t^h - 1
   n + 1 ≥ 2t^h
   (n + 1)/2 ≥ t^h
   h ≤ logₜ((n+1)/2)
   ```

**최종 공식**:
```
B-Tree 최대 높이 = ⌊logₜ((n+1)/2)⌋
                = O(logₜ n)
```

---

### 활동 3.2: 표에서 패턴 찾아 공식 추론

**데이터 분석**:

| n | t=2 | t=10 | t=100 | t=1000 |
|---|-----|------|-------|--------|
| 10³ | 9 | 3 | 2 | 1 |
| 10⁶ | 19 | 6 | 3 | 2 |
| 10⁹ | 29 | 9 | 5 | 3 |

**패턴 인식**:
```
[관찰 1] n이 10³배 → 높이가 log₁₀(10³) = 3 증가
[관찰 2] t가 10배 → 높이가 약 1/2로 감소

[일반화]
높이 ∝ log(n) / log(t) = logₜ(n)
```

---

### 활동 3.3: 작은 n값으로 공식 검증

**검증 예시** (t=3):

```
[공식] h ≤ log₃((n+1)/2)

n=10:  h ≤ log₃(5.5) ≈ 1.5 → h ≤ 1
실제: 루트에 5키, 자식에 5키씩 → 높이 1 ✓

n=20:  h ≤ log₃(10.5) ≈ 2.1 → h ≤ 2
실제: 검증 필요

n=100: h ≤ log₃(50.5) ≈ 3.5 → h ≤ 3
실제: 단계 2에서 검증 완료 ✓
```

## 학습 활동

### 활동 4.1: 귀납법으로 높이 상한 증명

**정리 (Theorem)**: 
> n개의 키를 가진 B-Tree(최소 차수 t)의 높이 h는 다음을 만족한다:
> ```
> h ≤ logₜ((n+1)/2)
> ```

**증명**:

**[보조정리] 높이 k인 B-Tree의 최소 키 수**

높이 k인 B-Tree의 최소 키 수 N(k)는 다음과 같다:
```
N(k) = 2t^k - 1  (k ≥ 0)
```

*귀납법으로 증명*:

**기본 (Base Case)**:
```
k = 0: N(0) = 1 = 2t^0 - 1 = 2 - 1 = 1 ✓
k = 1: N(1) = 1 + 2(t-1) = 2t - 1 = 2t^1 - 1 ✓
```

**귀납 가정 (Inductive Hypothesis)**:
```
높이 k-1인 B-Tree의 최소 키 수: N(k-1) = 2t^(k-1) - 1
```

**귀납 단계 (Inductive Step)**:
```
높이 k인 B-Tree:
- 루트: 1개 키
- 루트의 자식: 최소 t개
- 각 자식은 높이 k-1인 서브트리

N(k) = 1 + t × N(k-1)
     = 1 + t × (2t^(k-1) - 1)
     = 1 + 2t^k - t
     = 2t^k - t + 1
     ≥ 2t^k - 1  (t ≥ 1이므로)
```

**[주정리 증명]**:
```
n ≥ N(h) ≥ 2t^h - 1

n + 1 ≥ 2t^h
(n + 1)/2 ≥ t^h

h ≤ logₜ((n+1)/2)

∴ h = O(logₜ n)
```

---

### 활동 4.2: 수학적 표기법 익히기

**핵심 표기법 정리**:

| 표기법 | 의미 | B-Tree에서의 의미 |
|--------|------|------------------|
| O(f(n)) | 빅오 (상한) | 최악의 경우 시간 복잡도 |
| Ω(f(n)) | 빅오메가 (하한) | 최선의 경우 시간 복잡도 |
| Θ(f(n)) | 빅세타 (동차) | 평균적인 시간 복잡도 |
| ⌈x⌉ | 천장 함수 | 올림 |
| ⌊x⌋ | 바닥 함수 | 내림 |
| logₜ n | t를 밑으로 하는 로그 | 트리 높이 관련 |

**B-Tree 연산 복잡도**:
```
탐색(Search):  O(logₜ n)  [디스크 접근]
삽입(Insert):  O(logₜ n)  [디스크 접근]
삭제(Delete):  O(logₜ n)  [디스크 접근]
```

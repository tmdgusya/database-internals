## 2. B-Tree 핵심 개념

### 2.1 직관적 이해: 왜 B-Tree가 필요한가?

**문제 상황**: 디스크와 같은 외부 저장장치에서 데이터를 효율적으로 관리하려면?

- 디스크는 블록 단위로 데이터를 읽고 쓴다 (일반적으로 4KB)
- 이진 트리는 한 노드에 하나의 키만 저장 → 디스크 접근이 너무 잦음
- **해결책**: 한 노드에 여러 키를 저장하여 디스크 접근 횟수를 줄인다!

> B-Tree는 "팬아웃(fan-out)"이 큰 균형 트리로, 한 노드에 여러 키를 저장하여 디스크 I/O를 최소화한다.

### 2.2 구체적 예시: t=3인 B-Tree (2-3-4 Tree)

```
                    [30, 60]
                   /    |    \
            [10,20]  [40,50]  [70,80,90]
            /  |  \   /  |  \  /  |  |  \
           ...  ...  ... ... ... ... ... ...
```

**이 예시에서 관찰할 수 있는 것**:
- 루트 노드: 키 2개 (30, 60), 자식 3개
- 남부 노드: 키 2개 또는 3개
- 모든 리프 노드는 같은 깊이 (균형 트리!)

### 2.3 수학적 정의

#### 정의: 차수(order)가 $t$인 B-Tree

B-Tree $T$는 다음 **5가지 성질**을 만족하는 균형 탐색 트리이다:

| 성질 | 수학적 표현 | 의미 |
|------|-------------|------|
| **(1) 노드 키 개수 범위** | $t-1 \leq |keys| \leq 2t-1$ | 각 노드는 최소 $t-1$개, 최대 $2t-1$개의 키를 가진다 |
| **(2) 루트 특수 규칙** | $1 \leq |keys|_{root} \leq 2t-1$ | 루트는 최소 1개의 키를 가진다 |
| **(3) 자식 수 관계** | $|children| = |keys| + 1$ | $k$개의 키를 가진 노드는 $k+1$개의 자식을 가진다 |
| **(4) 키 순서** | $k_1 < k_2 < ... < k_m$ | 노드 내 키는 오름차순으로 정렬 |
| **(5) 균형 성질** | $\forall$ leaf: $depth(leaf) = h$ | 모든 리프 노드는 같은 깊이 $h$에 있다 |

#### 키 범위의 직관적 이해

```
차수 t = 3인 경우:
┌─────────────────────────────────────────┐
│  최소 키: t-1 = 2개                    │
│  최대 키: 2t-1 = 5개                   │
│  최소 자식: t = 3개                    │
│  최대 자식: 2t = 6개                   │
└─────────────────────────────────────────┘
```

**왜 $t-1$과 $2t-1$인가?**
- **최소 $t-1$개**: 노드가 너무 비어있지 않도록 보장 (공간 효율성)
- **최대 $2t-1$개**: 노드가 꽉 차면 분할(split) 발생 → 트리 성장

### 2.4 B-Tree의 높이 분석

#### 정리: B-Tree의 높이 상한

$n$개의 키를 저장하는 차수 $t$의 B-Tree의 높이 $h$는 다음을 만족한다:

$$h \leq \log_t \frac{n+1}{2}$$

#### 증명 과정 요약

**Step 1: 각 레벨의 최소 노드 수**

| 레벨 | 최소 노드 수 | 해당 레벨의 최소 키 수 |
|------|--------------|------------------------|
| 0 (루트) | 1 | 1 |
| 1 | 2 | $2(t-1)$ |
| 2 | $2t$ | $2t(t-1)$ |
| 3 | $2t^2$ | $2t^2(t-1)$ |
| $i$ | $2t^{i-1}$ | $2t^{i-1}(t-1)$ |

**Step 2: 전체 키 개수의 하한**

$$n \geq 1 + 2(t-1) \cdot \frac{t^h - 1}{t - 1} = 2t^h - 1$$

**Step 3: 높이 $h$에 대해 정리**

$$h \leq \log_t \frac{n+1}{2}$$

#### 직관과 수학의 연결

```
┌─────────────────────────────────────────────────────────────┐
│  높이 상한의 의미                                           │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  h ≤ log_t((n+1)/2) = (log_2((n+1)/2)) / (log_2(t))        │
│                                                             │
│  → t가 커질수록 (한 노드에 더 많은 키를 저장)               │
│    높이가 작아져서 검색이 빨라진다!                         │
│                                                             │
│  예시: n = 1,000,000 (100만 개 키)                         │
│  • t = 2 (이진 트리와 유사): h ≤ log_2(500000) ≈ 19        │
│  • t = 100: h ≤ log_100(500000) ≈ 3                        │
│                                                             │
│  → 디스크 접근 횟수가 19번 → 3번으로 줄어든다!             │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 2.5 연산의 시간 복잡도

| 연산 | 시간 복잡도 | 설명 |
|------|-------------|------|
| **검색** | $O(\log n)$ | 루트에서 리프까지 한 경로만 탐색 |
| **삽입** | $O(\log n)$ | 탐색 + 최대 $h$회 분할 |
| **삭제** | $O(\log n)$ | 탐색 + 재분배/병합 |

## 3. 수학적 검증

### 3.1 왜 최소 $t-1$개의 키가 필요한가?

#### 명제
> **명제 1.1**: B-Tree에서 각 노드가 최소 $t-1$개의 키를 가져야만 트리의 높이가 $O(\log n)$으로 보장된다.

#### 직관적 예상

**왜 $t-1$일까?**
- 키가 너무 적으면 → 트리가 한쪽으로 치우쳐서 **불균형**이 된다
- 키가 충분히 많으면 → 각 노드가 **충분한 자식**을 가져서 트리가 **넓게 펼쳐진다**
- $t-1$이라는 값은 "최소한의 안전장치" 역할을 한다

#### 증명 개요

**핵심 결과**: $n_{min} = 2t^h - 1$ → $h \leq \log_t(\frac{n+1}{2})$

| 레벨 | 최소 노드 수 | 각 노드의 최소 키 수 | 레벨별 최소 키 수 |
|------|-------------|-------------------|-----------------|
| 0 (루트) | 1 | 1 | 1 |
| 1 | 2 | $t-1$ | $2(t-1)$ |
| 2 | $2t$ | $t-1$ | $2t(t-1)$ |
| $i$ | $2t^{i-1}$ | $t-1$ | $2t^{i-1}(t-1)$ |

**총 최소 키 수** $n_{min} = 2t^h - 1$

### 3.2 왜 높이가 $O(\log n)$인가?

#### 명제
> **명제 2.1**: $n$개의 키를 저장하는 B-Tree의 높이는 $O(\log n)$이다.

#### 수학적 귀납법을 사용한 증명

**[보조정리] 높이 $k$인 B-Tree의 최소 키 수**

$$N(k) = 2t^k - 1 \quad (k \geq 0)$$

**귀납법 증명**:

**기본 (Base Case)**:
- $k = 0$: $N(0) = 1 = 2t^0 - 1 = 1$ ✓
- $k = 1$: $N(1) = 1 + 2(t-1) = 2t - 1 = 2t^1 - 1$ ✓

**귀납 가정**: 높이 $k-1$인 B-Tree의 최소 키 수: $N(k-1) = 2t^{k-1} - 1$

**귀납 단계**:
$$N(k) = 1 + t \times N(k-1) = 1 + t(2t^{k-1} - 1) = 2t^k - t + 1 \geq 2t^k - 1$$

**[주정리 증명]**:
$$n \geq N(h) \geq 2t^h - 1$$
$$n + 1 \geq 2t^h$$
$$h \leq \log_t((n+1)/2)$$

### 3.3 삽입/삭제 후에도 B-Tree 성질이 유지되는가?

#### 명제
> **명제 3.1**: B-Tree에 대한 삽입(INSERT)과 삭제(DELETE) 연산 후에도, B-Tree의 모든 성질이 유지된다.

#### 귀류법을 사용한 증명

**[가정]** 삽입/삭제 후 B-Tree의 어떤 성질이 깨졌다고 가정하자.

**케이스 1: 키 수 불변식 위반**
- 삭제 후 $t-1$개 미만 → 재분배/병합으로 해결 → 모순
- 삽입 후 $2t-1$개 초과 → 분할로 해결 → 모순

**케이스 2: 균형 불변식 위반**
- 모든 연산은 깊이를 유지 → 모순

**[결론]** 모든 경우에 모순이 발생하므로, 원래 가정은 거짓이다.

## 4. 실습 과제

### 레벨 1: 직관적 이해 (3개 과제)

#### 과제 1: t값에 따른 B-Tree 구조 분석

**난이도**: ⭐ (입문)

| t값 | 최소 키 수 | 최대 키 수 | 최소 자식 수 | 최대 자식 수 |
|:---:|:---:|:---:|:---:|:---:|
| t=2 | 1 | 3 | 2 | 4 |
| t=3 | 2 | 5 | 3 | 6 |
| t=4 | 3 | 7 | 4 | 8 |

**추가 질문 답변**:
1. **t=2인 B-Tree는 2-3-4 Tree**
2. **t값이 커질수록 트리 높이는 감소한다**

#### 과제 2: t값에 따른 트리 높이 변화 예측

n=20일 때:

| t값 | 최대 높이 (이론) | 실제 가능한 높이 |
|:---:|:---:|:---:|
| t=2 | 3 | 2 또는 3 |
| t=3 | 2 | 1 또는 2 |
| t=4 | 1 | 1 |

#### 과제 3: B-Tree와 이진 트리의 높이 비교

n=1000일 때:

| 자료구조 | 높이 |
|:---:|:---:|
| BST (최악) | 999 |
| BST (평균) | ≈ 14 |
| AVL/Red-Black | ≈ 14 |
| B-Tree (t=10) | ≤ 3 |
| B-Tree (t=100) | ≤ 2 |

### 레벨 2: 수치 검증 (3개 과제)

#### 과제 4: n=100, t=3일 때 최대 높이 계산

$$h_{max} \leq \log_3(50.5) \approx 3.57$$

**최대 높이 = 4** (레벨 기준)

#### 과제 5: 높이 h인 B-Tree의 키 개수 범위

**최소 키 수**: $n_{min} = 2t^h - 1$

**최대 키 수**: $n_{max} = (2t)^{h+1} - 1$

t=3, h=3일 때: $53 \leq n \leq 1295$

#### 과제 6: 분할 연산 조건의 수식화

**분할 조건**: 노드의 키 개수 $\geq 2t - 1$

### 레벨 3: 수식 유도 (3개 과제)

#### 과제 7: 높이 상한 공식 유도

$$h \leq \log_t\frac{n+1}{2}$$

**증명 단계**:
1. 최소 키 수: $n_{min}(h) = 2t^h - 1$
2. $n \geq 2t^h - 1$
3. $h \leq \log_t((n+1)/2)$

#### 과제 8: 키 개수 n과 노드 수 N의 관계

**최소 노드 수**: $N_{min} = \lceil \frac{n}{2t-1} \rceil$

**최대 노드 수**: $N_{max} = \lfloor \frac{n+t-2}{t-1} \rfloor$

#### 과제 9: 삽입 연산 $O(\log n)$ 증명

- 탐색: $O(h) = O(\log n)$
- 분할: 최대 $h$회, 각 $O(t)$
- 총: $O(t \cdot \log_t n) = O(\log n)$ (t가 상수)

### 레벨 4: 확장 응용 (3개 과제)

#### 과제 10: B+Tree와 B-Tree 수학적 차이

- B+Tree: 모든 데이터가 리프에 저장
- 남부 노드: 인덱스만 저장 → 더 많은 키 저장 가능
- 범위 쿼리: 리프 연결 리스트로 $O(k)$ 시간

#### 과제 11: B*Tree 채움 비율 변화

- B*Tree: 최소 채움 비율 2/3
- 최소 키 수 증가 → 공간 효율성 향상

#### 과제 12: 실제 DB 인덱스 높이 예측

**시나리오**: 10억 레코드, 4KB 블록, 8바이트 키/포인터

- 적절한 t값: 약 128
- 예상 최대 높이: $\log_{128}(10^9) \approx 4.3$ → **최대 5**
